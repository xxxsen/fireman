# 计划设置、调仓工作台与持仓校正

- 更新：2026-06-14

## 概述

计划内主路径已收敛为：

```text
全部计划
  → 进入单个计划
  → 组合总览
  → 调仓工作台
  → 持仓校正
  → 计划设置
```

本轮调整的目标不是单纯美化，而是统一计划内的信息架构，明确“模板管理”“当前计划配置”“真实持仓录入”“调仓规划”四类职责的边界。

## 正式信息架构

### 全局导航

左侧侧边栏当前为：

- 计划
- 资产资料库
- 配置模板
- 设置

其中：

- `配置模板` 是全局模板管理入口，对应 `/scenarios`
- 资产资料库负责资产检索、资料录入、抓取状态查看与删除态控制

### 计划内导航

单计划页签当前为：

| 路由 | 标签 | 职责 |
| --- | --- | --- |
| `/plans/{id}/overview` | 组合总览 | 查看计划概况、结构偏离、关键入口 |
| `/plans/{id}/rebalance` | 调仓工作台 | 查看当前持仓与目标结构，进入持仓校正、调仓计划或调仓执行 |
| `/plans/{id}/settings` | 计划设置 | 切换当前计划使用的配置模板、编辑计划参数、运行模拟 |

已下线并兼容跳转：

- `/plans/{id}/holdings` → `/plans/{id}/rebalance`
- `/plans/{id}/scenarios` → `/scenarios`

### 计划上下文栏

顶部上下文栏当前固定为：

```text
全部计划 / 当前计划名称
```

不再提供：

- 计划切换下拉
- 计划详情页内的新建计划按钮

## 计划设置

`计划设置` 当前只负责当前计划，不承担全局模板编辑。

页面包含三部分：

1. 目标配置
2. FIRE 参数
3. 模拟与分析

### 目标配置

- 用户可以在当前计划内切换使用哪个配置模板
- 大类目标与地区组内目标在此处只读展示
- 如需修改模板结构，必须跳转到 `/scenarios`

这保证了：

- 模板编辑发生在全局模块
- 计划页只负责“当前计划引用哪个模板”

### FIRE 参数

- `计划名称` 可直接编辑
- 其余 FIRE 参数沿用现有编辑方式
- 保存条与离开提示都基于真实 dirty 对比

当前规则：

- 用户点击但未改值，不显示保存按钮
- 表单恢复到初始值后，dirty 自动清空
- 只有真实存在未保存变更时，页面切换才会触发离开确认

## 调仓工作台

`/plans/{id}/rebalance` 是计划内调仓相关主入口，并收拢了原持仓管理页的主职责。

当前页面特征：

- 按大类 → 地区 → 标的展示当前持仓与目标结构
- 目标地区权重为 `0`、但当前仍持有的资产，仍必须展示
- 资产名称可跳转到资产资料库详情
- 零值差额不再显示 `待投入 ¥0.00` / `待减配 ¥0.00`

当前主操作：

- 持仓校正
- 创建 / 继续调仓计划
- 调仓执行

已移除的低价值动作：

- 动作筛选
- 导出 CSV
- 按计划规模对齐（高级折叠区）
- 记录调仓后快照
- 行内“在持仓中编辑”

## 持仓校正

`/plans/{id}/asset-refresh` 已成为计划内唯一的“更新真实持仓结构与金额”入口，保留原路由但用户可见文案统一为“持仓校正”。

### 任务定义

该流程用于：

- 新增标的
- 移除标的
- 修改组内配比
- 修改当前金额
- 查看当前配置模板（只读）；如需切换，前往计划设置

不再使用独立的“启用 / 停用”模型。

### 提交流程

最终提交已收敛为一次后端原子事务，统一完成：

1. 切换目标方案
2. 更新持仓结构
3. 更新当前金额
4. 同步计划总资产
5. 写入审计事件

这保证了：

- 成功时整笔提交一起生效
- 失败时不会留下“部分已提交”的半状态

### 关键交互规则

- `确认范围` 中展示的是当前计划内的配置模板，不在流程内切换模板
- `录入当前资产` 中，资产行可编辑金额与组内配比
- `添加标的` 使用居中的模态窗口，不使用右侧抽屉
- 资产总值的币种与数量级提示内联展示，如 `CNY(万) 15000`
- 单位提示只表示当前输入值已达到对应数量级，不做换算改写
- `影响资产数量` 只统计本次真实发生变化的资产
- 若变更前后总资产一致，确认页会明确提示“本次变更未改变资产总值”
- “同步计划基准规模至新总值” 复选框已取消，持仓校正默认同步当前计划基准规模

## 资产资料库补充约定

与本轮收拢一并落地的资料库交互规则：

- 列表页 `操作` 列提供删除按钮
- 已被计划引用的资产，删除按钮置灰并给出原因提示
- `查看抓取状态` 使用居中模态窗口
- 抓取任务完成后，列表与详情会自动刷新，避免用户误判任务卡住
