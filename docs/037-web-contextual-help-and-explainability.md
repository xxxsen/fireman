# Web 上下文帮助与计算可解释性

## 1. 目标与边界

Fireman Web 在用户做输入选择、阅读计算结果和比较图表时，就地回答以下问题：

1. 这个概念是什么，输入会改变什么；
2. 结果使用哪份计划、模拟或历史数据；
3. 哪些数据在计算期间保持不变；
4. 数值如何计算或判定；
5. 数值高低和边界状态应如何解读；
6. 结果不能推出哪些保证；
7. 下一步会修改什么，是否需要重新计算。

这套交互只解释既有业务口径，不修改 FIRE、快算、模拟、回测、优化、压力测试或敏感性测试的公式，也不引入新的 UI 依赖。

## 2. 帮助信息模型

固定定义集中在 `web/lib/terms.ts` 的 `HELP_TOPICS`。每个 topic 至少包含中文 `label` 与一句话 `summary`；经过聚合、折算、模拟、离散搜索或风险调整得到的计算型 topic 还必须包含：

- `calculation`：用户可读的公式或算法口径；
- `inputs`：数据来自当前输入、冻结模拟、历史行情还是计划持仓；
- `interpretation`：数值方向、边界状态与不能得出的结论。

可选的 `purpose` 和 `caveat` 分别描述用途与限制。页面可以补充本次运行的动态值，但不得复制或另写与注册表冲突的固定定义。

topic key 使用稳定业务概念命名，不绑定页面。相同中文标签不得指向不同定义；模拟统计与历史回测等口径不同的指标使用明确标签区分。注册项必须在非测试界面至少挂载一次；不存在业务入口的概念应删除，而不是长期保留孤儿定义。

## 3. 共享组件

### 3.1 `MetricHelp` 与 `HelpLabel`

单个字段、表头、指标或状态旁使用 `MetricHelp`。`HelpLabel` 负责组合可见标签与帮助入口，供原生 `label`、`dt`、`th`、标题和指标卡复用。

帮助浮层按已有字段依次展示“作用、计算、数据、解读、注意”。触发器具备稳定的中文 accessible name，点击区域不小于 32×32px；鼠标 hover、键盘 focus 和点击/触屏均可打开。再次点击、Escape 或点击外部关闭，键盘关闭后焦点回到触发器。

未知 topic 在开发和测试环境抛错，避免界面静默失去解释；生产环境安全降级。帮助内容不放链接、表单或必须二次操作的控件。

### 3.2 `CalculationExplanation`

结果级说明使用页面内的 `CalculationExplanation`，而不是把完整计算逻辑塞进窄浮层。默认显示“这次到底计算了什么”和一句业务结论，展开后依次展示：

- 回答的问题；
- 改变的内容；
- 保持不变的内容；
- 本次使用的数据和版本；
- 计算与判定；
- 限制与不确定性；
- 下一步。

engine、algorithm、hash 等审计字段进入“高级计算详情”，不压过普通用户首先需要的结论。FIRE 正式模拟、计划改善器、达标前沿和快算均遵循这一结构。

### 3.3 `ChartFrame`

ECharts 和自绘 SVG 图使用 `ChartFrame` 或遵循同等约束：图外可见标题、横轴、纵轴、单位、图例语义和“如何解读”；必要时提供使用同一数据源的数据表。Canvas/SVG 自身仍启用 aria 描述，关键结果不能只存在于像素中。

## 4. 页面覆盖

### 4.1 FIRE 主链路

- 新建计划与计划参数解释现金流、目标配置、提款策略、通胀过程、税率、调仓、收益假设、Profile、路径数和 seed。
- Monte Carlo 结果区区分点估计与 Wilson 区间，解释分位数、代表路径、名义/真实金额、回撤、失败年龄、恢复期、冻结输入和结果过期。
- 计划改善器明确只搜索退休年龄、储蓄、退休支出和稳定收入四类现金流杠杆；候选使用同 seed 与 path number 配对，Wilson 下界达标才列为稳健方案。
- FIRE 达标前沿的四个问题各有独立的“作用、改变、保持、计算”帮助，结果展示来源模拟、冻结现金流/持仓、路径、seed、离散搜索域、相邻不利点和 Wilson 判定。
- FIRE 快算明确采用固定收益与确定性通胀，不读取持仓或市场历史，不输出成功概率；所需 FIRE 资本来自逐月反推，而不是年支出除以收益率。

默认目标成功率为 95%；来源正式模拟路径足够时，默认评估路径为 10,000 条。

### 4.2 研究、假设与资产

- 回测解释共同区间、前值填充、复权、再平衡、交易费、CAGR、波动率、回撤、Sharpe、Calmar、VaR/CVaR、滚动指标、贡献和相关性。
- 单资产投入路径结果解释累计投入、期末资产、XIRR、时间加权收益、单位净值回撤、本金浮亏时间、交易成本与滚动分位数；账户价值图显示日期和金额坐标轴，并为鼠标、触摸及键盘提供单日点位详情。
- 自动调优结果可见说明当前页签的主排序、约束和稳定并列规则；尾部风险排序不被描述为更高年化收益。
- 模拟假设 Profile 解释 `ν`、`μ/σ/φ`、收益/FX 位移、波动率乘子、先验强度、相关性 `ρ`、最小特征值和 PSD 修复。
- 资产目录、资产详情和模拟 readiness 解释资产/数据类别、净值主代码、费用、data-as-of、复权、行情点类型、完整年度、模拟快照及同步影响。

### 4.3 持仓、调仓与管理后台

- 计划总览、持仓校正和调仓工作台统一解释持仓市值、计划基准规模、目标/当前权重、组内配比、结构偏差、颜色语义与需调仓标的。
- 管理后台解释 worker type、scope、claim、attempt、heartbeat、lease、pre-complete、finalization、重试耗尽、数据版本和同步健康。
- 完整 ID、URL 和审计字段可以提供完整值查看；状态机制、禁用原因和业务副作用不得只依赖原生 HTML `title`。

## 5. 图表交互不变量

每个图表必须满足：

1. 横轴、纵轴和单位以可见文字展示，纵轴标题与刻度预留足够空间；
2. 图例使用业务名称，不暴露内部 series key；
3. 鼠标移入点、柱、单元格或扇区时显示完整详情，触屏点按具有等价入口；
4. 悬浮信息包含横轴值、业务序列名、格式化数值和必要的比较基准；
5. 缺失值显示“无数据”或“历史不足”，不得格式化为零；
6. 平滑、连线、分位带、归一化和颜色的视觉含义在图旁说明；
7. 达标/失败、正/负和目标/当前不能只用颜色区分；
8. 375px 宽度下不产生页面横向滚动，浮层保持在视口内。

FIRE 前沿年龄图的横轴为退休年龄（岁），纵轴按问题分别为最大达标年度退休支出或最低达标首年储蓄。点 hover 或键盘聚焦显示年龄、金额、边界状态、成功路径、点估计、Wilson 区间、目标判定和相邻档位。浮层纵向固定在图内安全区域，横向随点对齐，避免顶部/底部点将详情推离视口。

图表 tooltip formatter 使用可导出的纯函数。单元测试覆盖首点、末点、负值、缺失值和大金额；数据表、摘要和图表不得各自重复计算口径。

## 6. 错误、禁用与过期状态

禁用操作旁必须有可见原因或可访问 tooltip。stale、搜索域内无解、整个搜索域均达标、同步未就绪和任务失败都要说明用户可执行的下一步。

服务端业务错误由前端展示可读消息、稳定错误码和 `request_id`；未知内部错误不能泄漏 SQL 或堆栈。页面不得只显示无法关联日志的 `internal server error`。

## 7. 自动化门禁

术语测试持续验证：

- topic 的 label/summary 完整；
- 计算型 topic 的 calculation/inputs/interpretation 完整；
- 无未挂载 topic；
- 同名标签无冲突定义；
- FIRE、研究和假设高风险页面仍挂载对应帮助。

组件测试覆盖 hover、focus、click/tap、Escape、外部点击、焦点恢复、未知 topic 和结构化内容。页面与图表测试覆盖输入解释、结果解释、轴单位、悬浮详情、缺失/负值/大金额和移动布局。

前端变更的交付门禁为：

```bash
make web-install
make web-lint
make web-test
make web-build
```

涉及后端契约或计算逻辑时，再按影响范围运行后端 build、test、lint 和 integration test。
