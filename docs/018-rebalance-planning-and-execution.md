# 调仓计划、参考方案与多日执行

## 目的

Fireman 将“生成调仓建议”和“登记真实执行”拆成两个层次：

- 调仓计划负责根据当前持仓、目标配置和现金池生成参考方案。
- 调仓执行负责分多日登记卖出、买入、备注和最终写回。

这种分层保证计划可以反复推演，而真实执行有独立状态、时间线和现金池约束。

## 调仓计划

调仓计划以当前计划快照为输入，生成参考调仓方案：

- 目标金额来自计划目标配置和当前总资产规模；
- delta 表示每个持仓与目标金额的差额；
- 买入/卖出建议不直接修改持仓，必须由用户确认；
- 行级“应用推荐金额”一次只应用一行，避免批量误操作；
- 未分配资金在提交时归集到现金持仓。

计划草稿保存 reference package 和用户编辑后的行级动作，便于回到页面继续调整。

## 现金池

现金池是一等公民：

- 卖出登记先进入现金池；
- 买入只能消耗现金池可用余额；
- 完成执行时，剩余未分配资金归集到现金资产；
- 现金支出不计交易成本，买卖调仓按交易方向和金额计算费用。

## 多日调仓执行

调仓执行是过程对象，而不是一次性表单。执行状态包括草稿/进行中/已完成/已取消，并维护：

- 执行行：每个标的的目标、计划动作、已执行数量和剩余差额；
- 执行事件：卖出、买入、备注、完成、取消；
- 执行摘要：现金池余额、已完成比例、未处理行。

同一计划在存在 active 执行时，不允许创建互相冲突的新执行；持仓校正与调仓执行也需要在入口上保持清晰分层。

## 页面约定

- `/plans/{id}/rebalance` 是调仓工作台，展示当前持仓、目标结构、偏离摘要，并提供持仓校正、创建/继续调仓计划和调仓执行入口。
- `/plans/{id}/rebalance/plan/{draftId}` 展示参考方案、行级推荐和提交入口。
- `/plans/{id}/rebalance/executions` 展示执行列表。
- `/plans/{id}/rebalance/executions/{executionId}` 作为执行工作区，登记卖出、买入、备注、完成或取消。

持仓校正页面仍用于调整计划持仓结构与金额；调仓执行页面只处理真实交易过程。

## 数据一致性

- 创建草稿、提交草稿、登记执行事件和完成写回都必须在事务中完成。
- 完成执行时重新校验计划版本和执行状态，防止过期页面覆盖新数据。
- 完成后写回持仓，并保留执行时间线用于复盘。
- 取消执行不回滚已写入的历史事件，但不会写回持仓。

## 验证重点

- 参考方案 delta、行级应用推荐和 cash sweep 的金额计算。
- 现金池余额不能为负，买入金额不能超过可用现金。
- 完成执行后持仓金额、现金持仓和执行状态一致。
- 活跃执行与持仓校正/新执行之间的互斥规则。
