# 资产变更与调仓计划（用户路径）

> 本文概述当前已实现的用户路径与提交流程；页面职责总览请同时参考 [008-计划设置、持仓预览与资产变更](./008-plan-settings-holdings-preview.md)。

## 日常更新市值

持仓预览 / 组合总览 → **资产变更** → 录入分项 + 总值 + 组内配比 → 确认

资产变更提交后，当前计划总资产默认同步到本次确认后的新总值。

## 中途改结构

资产变更 → 切换当前计划使用的 FIRE 方案 / 新增或移除标的 / 调整组内配比 / 录入当前金额 → 确认

## 结构偏移修正

持仓预览 → 查看结构偏差 → **创建调仓计划**

→ 查看 **参考调仓方案**（完整对齐变动，含未达阈值的微调）

→ 逐行 **应用推荐金额** 或手工改计划当前金额（看资金池）

→ 分阶段暂存 / 撤销

→ 完成并更新持仓（未分配资金默认计入现金，或接受组合规模下降）

### 参考调仓方案 vs 行内建议

| | 行内「参考建议」 | 参考调仓方案 |
| --- | --- | --- |
| 阈值 | 受调仓阈值影响，未达阈值显示「不动」 | 不受阈值过滤 |
| 操作 | 无自动应用 | 每行可「应用推荐金额」（一次一行） |
| 冻结 | 创建时冻结 | 创建时冻结 `recommended_package_delta_minor` |

## 与旧流程的区别

- 独立 `持仓管理` 页已下线，真实持仓编辑统一收拢到 `资产变更`
- 调仓计划草稿持久化在数据库，刷新页面或换设备可继续编辑
- 调仓计划编辑时冻结结构目标/还差，仅资金池反映增减配平衡
