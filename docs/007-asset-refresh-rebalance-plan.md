# 持仓校正与调仓执行（用户路径）

> 本文概述当前已实现的用户路径与提交流程；页面职责总览请同时参考 [008-计划设置、调仓工作台与持仓校正](./008-plan-settings-holdings-preview.md) 与 [020-web-ui-information-architecture-and-accessibility.md](./020-web-ui-information-architecture-and-accessibility.md)。

## 日常更新市值

调仓工作台 / 组合总览 → **持仓校正** → 录入分项 + 总值 + 组内配比 → 确认

持仓校正提交后，当前计划基准规模默认同步到本次确认后的新总值。

## 中途改结构

持仓校正 → 查看当前配置模板 / 新增或移除标的 / 调整组内配比 / 录入当前金额 → 确认

## 结构偏移修正

调仓工作台 → 查看结构偏差 → **创建调仓执行**

→ 创建时冻结每行目标变动额（不受调仓阈值过滤）

→ 分多日登记真实卖出 / 买入事件（卖出先归集资金池，买入从资金池支出）

→ **Complete** 按实际执行增减推导最终持仓，剩余资金自动归集现金，单事务落库

详见 [018-rebalance-planning-and-execution.md](./018-rebalance-planning-and-execution.md)。

## 与旧流程的区别

- 独立 `持仓管理` 页已下线，真实持仓编辑统一收拢到 `持仓校正`
- 「调仓计划（draft）」链路已整体下线（相关表不在当前完整 schema 中）：
  把持仓从当前结构调到目标结构用**调仓执行**；把持仓事实改成任意状态用**持仓校正**
