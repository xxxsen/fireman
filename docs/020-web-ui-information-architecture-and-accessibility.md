# Web UI 信息架构、术语与可访问性规范

## 目的

Fireman Web 端已经形成一套稳定的信息架构与组件规范。本文整理当前已落地的页面职责、术语、向导结构、调仓主链路、任务反馈和可访问性约定，作为 `docs/011`、`docs/012`、`docs/018` 的补充。

## 计划内信息架构

单个计划内的主导航固定为三个页签：

| 路由 | 标签 | 职责 |
| --- | --- | --- |
| `/plans/{id}/overview` | 组合总览 | 查看计划概况、配置图表、关键偏离和主入口 |
| `/plans/{id}/rebalance` | 调仓工作台 | 查看当前持仓与目标结构，进入持仓校正、调仓计划或调仓执行 |
| `/plans/{id}/settings` | 计划设置 | 管理目标配置、FIRE 参数、模拟与分析 |

`/plans/{id}/asset-refresh` 是调仓工作台的子流程，页签仍归属“调仓工作台”。深层页面使用 `PlanPageHeader` 统一标题、描述和操作区。

## 术语口径

| 概念 | 统一用词 | 说明 |
| --- | --- | --- |
| 全局资产配置权重模板 | 配置模板 | 对应 `/scenarios`，不再称为“FIRE 方案”或“场景模板” |
| 收益假设三档 | 假设情景 | 保守 / 基准 / 乐观，仅用于模拟假设 profile |
| 压力测试冲击设定 | 压力场景 | 仅出现在压力测试结果中 |
| `total_assets_minor` | 计划基准规模 | 表单短标签可写为“基准规模” |
| `/plans/{id}/rebalance` | 调仓工作台 | 不是只读预览，而是调仓相关主入口 |
| `/plans/{id}/asset-refresh` | 持仓校正 | 更新真实持仓结构与金额，路由保持不变 |

页面文案和术语提示以 `web/lib/terms.ts` 为集中口径。`MetricHelp` 的 aria label 使用术语中文名，屏幕阅读器可区分不同提示点。

## 调仓主链路

调仓工作台同时提供三类入口：

- `持仓校正`：更新真实持仓结构与金额；
- `创建 / 继续调仓计划`：进入调仓计划草稿，逐行应用参考方案；
- `调仓执行`：创建多日执行单，登记真实卖出、买入、备注、完成或放弃。

创建调仓计划和调仓执行前必须二次确认。存在 active 调仓执行时，持仓校正和新调仓计划入口禁用，并提示用户继续或放弃当前执行。

调仓工作台加载时必须确认 active 调仓计划草稿状态。若无法确认且没有可用缓存，页面应进入错误态并提供重试，不得提前展示“创建调仓计划”或“调仓执行”等会产生新过程对象的入口；已有可用草稿数据时，可继续渲染“继续调仓计划”。

## 向导与表单

新建计划和持仓校正共用 `Stepper` 步骤条。主要提交/导航操作使用 `Button`，基础尺寸为 `min-h-10`，移动主操作可使用 `size="lg"`。

持仓校正流程为三步：

1. 确认范围：说明本次覆盖范围，并只读展示当前配置模板；
2. 录入当前资产：新增、移除、调整金额和组内配比；
3. 确认提交：展示变更摘要并提交。

配置模板切换不再埋在持仓校正流程里，用户需前往计划设置修改。

## 模拟与分析反馈

组合总览对模拟 job 使用显式状态分支：

- `pending/running`：显示进度和计划设置入口；
- `failed`：显示失败原因和重试入口；
- `succeeded`：显示完成提示；
- `canceled`：不保留“正在运行”提示。

分析中心的 Monte Carlo、压力测试和敏感性测试各自维护忙碌态和错误态。附属分析需要选中的 Monte Carlo run 已完成；否则按钮禁用并显示原因。

## 模拟假设页

模拟假设页拆分为列表、默认偏好、详情和编辑器组件。profile 详情加载失败时必须显示 `ErrorState` 和重试入口。

编辑器中资产大类、地区和币种使用受控枚举选择；系统 profile 只读，复制为用户 draft 后才能编辑。

## 移动端与可访问性

- 高频宽表在移动端使用卡片形态：调仓工作台汇总、调仓计划行、调仓执行列表和执行行。
- `Dialog` 打开时锁定背景滚动，关闭后恢复。
- `tablist` 需要配对 `tabpanel` 与 `aria-labelledby`，并支持左右键切换。
- 页面加载优先使用骨架或明确 loading 状态，根容器统一使用 `content-enter`。

## 验证重点

- 调仓计划草稿入口可达，创建/继续路径完整。
- 调仓执行创建有确认，active 执行会禁用互斥入口。
- 模拟 job failed 不再显示“正在后台运行”。
- 模拟假设详情断网显示错误态。
- 375px 宽度下调仓主链路无横向滚动即可完成主要操作。
